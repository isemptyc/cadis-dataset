# 3. Dataset Scope

The Brazil release scope is full Brazil ISO2 administrative coverage represented by the selected Geofabrik Brazil PBF and the Natural Earth `BR` country boundary.

The build and evaluation both used the same scoped boundary generated by:

* `build_br_boundaries.py`

Boundary selection rule:

* Union OSM administrative relation areas at level 4 whose representative point is covered by the Natural Earth `BR` boundary.

Generated scope evidence:

* Boundary geometry type: `MultiPolygon`
* Scope bbox: `[-73.9830625, -33.8694284, -28.6289646, 5.2695808]`
* Included level-4 units: `27`

No intentional sovereign-territory subset policy was applied.

---

# 4. Administrative Model

Brazil is modeled with the following runtime levels:

| Level | Label |
| ----- | ----- |
| `4` | State / Federal District |
| `8` | Municipality |
| `9` | District / submunicipal unit |

Level `10` was intentionally excluded from this initial release. The source PBF contains many level-10 neighborhood-style features, but the probe showed substantial parent incompleteness at that level. Excluding level `10` keeps the initial dataset focused on stable state, municipality, and district hierarchy.

Final staged dataset structural counts:

| Level | Count |
| ----- | ----: |
| `4` | 27 |
| `8` | 5,602 |
| `9` | 6,721 |

The dataset-scoped hierarchy contains `12,350` nodes and `12,318` parent edges.

# 5. Test Methodology

## 5.1 Sampling Strategy

* Total samples: `50,000`
* Sampling mode: mixed inside/outside stress testing
* Inside samples: `45,000`
* Outside samples: `5,000`

The test intentionally injects out-of-country points to validate:

* Boundary rejection behavior
* Offshore classification
* Policy-layer containment
* Cross-border isolation

Sampling is uniform over the scoped boundary area, not population-weighted.
