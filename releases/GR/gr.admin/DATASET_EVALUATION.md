# 3. Dataset Scope

The dataset scope is full ISO2 Greece administrative coverage.

The build uses a Natural Earth Greece admin-0 boundary to prevent neighboring
country administrative relations from entering the Greece dataset. This is
required because the unfiltered Greece PBF contains nearby foreign
administrative relations.

Evaluation uses a derived OSM administrative coverage boundary generated by:

`build_gr_boundaries.py`

Boundary outputs:

| Boundary | Path | Purpose |
| --- | --- | --- |
| Build boundary | `gr_country.json` | Natural Earth admin-0 Greece scope filter |
| Evaluation boundary | `gr_admin_coverage_boundary.json` | Union of scoped OSM admin polygons at levels 3,4,5,6,7,8, buffered inward by `0.00005` degrees |

The evaluation boundary avoids Natural Earth maritime and coarse-island
false-inside samples while still testing the released administrative coverage.
The tiny inward buffer avoids sub-meter boundary precision slivers between the
source polygon model and the runtime FFSF geometry encoding.

Special modeling note:

* `admin_level=3` is included for the Autonomous Monastic State of the Holy Mountain.
* Core Greece administrative output uses levels `4,5,6,7,8`.

# 4. Test Methodology

## 4.1 Sampling Strategy

* Total samples: `50,000`
* Sampling mode: mixed inside/outside stress testing
* Inside ratio: `0.9`
* Expected outside ratio: `0.1`
* Evaluation workers: `1`

Single-worker evaluation is intentional. The threaded evaluator path undercounts
results in this checkout; `--evaluate-workers 1` is required for reliable
aggregation.

The run intentionally injects about 10% outside-boundary points to validate:

* boundary rejection behavior
* offshore handling
* nearby fallback containment
* cross-border isolation
