# 3. Dataset Scope

The dataset scope is full ISO2 Turkey administrative coverage.

The build uses a Natural Earth Turkey admin-0 boundary to prevent neighboring
country administrative relations from entering the Turkey dataset. The scoped
hierarchy probe reduced unfiltered neighboring leakage and produced the core
Turkey model:

| Admin Level | Meaning | Scoped Probe Count |
| --- | --- | ---: |
| `4` | Province | 78 |
| `6` | District | 970 |
| `8` | Neighborhood / village-level local unit | 13,329 |

The engine intentionally excludes sparse `5`, `7`, `9`, and `10` levels from
the first release contract. Level `7` exists in the source data, but is sparse
municipal detail and is not required for the stable province/district/local-unit
runtime model.

Evaluation uses a derived OSM administrative coverage boundary generated by:

`build_tr_boundaries.py`

Boundary outputs:

| Boundary | Path | Purpose |
| --- | --- | --- |
| Build boundary | `tr_country.json` | Natural Earth admin-0 Turkey scope filter |
| Evaluation boundary | `tr_admin_coverage_boundary.json` | Union of scoped OSM admin polygons at levels 4,6,8, buffered inward by `0.00005` degrees |

The evaluation boundary avoids Natural Earth maritime/coastal false-inside
samples while still testing the released administrative coverage. The tiny
inward buffer avoids sub-meter boundary precision slivers between the source
polygon model and the runtime FFSF geometry encoding.

# 4. Test Methodology

## 4.1 Sampling Strategy

* Total samples: `50,000`
* Sampling mode: mixed inside/outside stress testing
* Inside ratio: `0.9`
* Expected outside ratio: `0.1`
* Evaluation boundary: derived Turkey OSM administrative coverage boundary
* Workers: `1`

The test intentionally injects approximately 10% out-of-country points to
validate boundary rejection behavior, offshore classification, policy-layer
containment, and cross-border isolation.

Sampling is uniform over administrative land coverage, not population-weighted.

# 11. Boundary Isolation Validation

Under stress testing with 10% forced out-of-bound samples:

* No boundary-leak failure was observed in this sampled run.
* Expected outside samples resolved to empty-shape or offshore outcomes.
* Nearby fallback did not create cross-border escalation.
* Evaluation against Natural Earth directly produced 8 false inside failures
  near coastal/boundary areas; the reproducible OSM administrative coverage
  evaluation boundary removes those sampling artifacts.

This confirms strict boundary containment within the staged TR dataset.

---

# 12. Structural Observations

1. Turkey can be represented by a stable `4,6,8` runtime model for the first release.
2. Sparse source levels `5`, `7`, `9`, and `10` are not needed for the initial
   release contract.
3. Geometry integrity is high under the scoped OSM administrative coverage boundary.
4. Parent linkage incompleteness exists but does not cause coverage failures in
   the sampled policy run.
5. Nearby fallback is minimal and bounded.
6. No explicit semantic repair layer is required.
